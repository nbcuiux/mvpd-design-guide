Discovery Script

We prepared a script to understand the current MVPD Admin system, users’ current workflow, pain points, and needs. We started with general topics to prompt users to talk about how they use and feel about their current system, and followed up with more specific questions as users went through the workflows.

MVPD Introduction

MVPD is Multi-Video Platform Distributor. Users from NBC Digital Distribution (DD) use this Admin tool to add and update MVPD records every month, and manage brand entitlements.

We conducted three discovery sessions to prompt knowledgeable users to talk about their daily work in the current MVPD Admin system, including the system’s functions, their main workflows, and current pain points.

In Discovery Session 1, the user introduced that this MVPD Admin System is an authentication and authorization tool to manage MVPD records and their entitlements. It is also a database for everybody to track data. Users also gave a brief overview of the records living in the system.

MVPD Management Process

This process showed the MVPD management process by NBC DD, MVPD partner, and Adobe Prime Time.

Adding New MVPD Records

In Discovery Session 2, the user demonstrated some main important workflows, including collecting information from partner, entering new MVPD records into the Admin system, managing brand entitlements, and managing whitelist groups.

In the Follow Up session, we discussed and clarified a few topics based on Discovery session 2, including required data fields, logo management, MVPD Transfer, entitlement cloning, and optimizing operations on whitelist groups. The user also talked about ways to manage single MVPDs, and aggregator MVPDs.

Reducing the Number of Clicks

In Discovery Session 3, we focused on some tasks in which we could possibly reduce the number of clicks in the new design. The tasks are: MVPD transfer, cloning entitlements (in the context of creating a brand new record), management of whitelist groups, decommission a brand, and adding a new platform.

Pain Points

  • There are 400+ entries of MVPD Records living in the system. Every week, the MVPD admin needs to add new records into the system manually.
    • Currently, users collect new information from MVPD from Excel sheets and emails. It is very time-consuming.
    • After receiving new information, users need to enter every single record by hand.
  • When managing the MVPD records in the system, the bulk actions are very restricted.
    • Managing brand entitlements requires too many clicks in check boxes. Need to make selections for 24 brands on 11 platforms and 2 instances one by one.
    • Some workflows need to be optimized, e.g. logo management, transfers record, and decommission a brand.

Wants

  • Provide MVPD partners direct access to the tool with limited capability, so that they can enter information on their own
  • Provide bulk upload functions for DD admin and MVPD partners to quickly import data into the system
  • Optimize current workflows and create new workflows to augment bulk edit action, for more efficient management
