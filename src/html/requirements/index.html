MVPD Requirements

The requirements include two key scope areas:

  • Redesign the MVPD management workflow to allow role-based access and bulk upload
  • Reducing the number of clicks during tasks

Re-design current Onboarding workflows to enhance the MVPD Admin Tool, allowing for a strategic streamline for partners, vendors, and internal personnel.

Single MVPD Management

Refactor Single MVPD Management Workflow:

1. Use Drupal’s native Roles and Permissions features to provide MVPDs direct access to the tool but with limited capability to enter the following sections:

  • Basic Information
  • Support Information
  • MVPD Logos
  • Test Credentials
2. Refactor MVPD Entitlements Section to greatly reduce the number of clicks for DD Ops to entitle an MVPD quickly across brands and platforms

Bulk upload creation

Build Bulk MVPD Creation Capability:
  • Agree on xlsx template format and required fields
  • Build Bulk MVPD Import Capability that will populate Basic Information, Support Information and Test credentials fields per MVPD
  • DD Ops will need to manually upload logo and generate, as well as perform all entitlements work (via new streamlined workflow per above)
